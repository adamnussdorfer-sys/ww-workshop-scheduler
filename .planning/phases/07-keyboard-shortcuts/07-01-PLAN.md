Create two new files:

**1. `src/hooks/useKeyboardShortcuts.js`**

Create directory `src/hooks/` if it does not exist.

Export a named function `useKeyboardShortcuts({ onPrevWeek, onNextWeek, onToday, onClosePanel, onNewWorkshop, isPanelOpen })`.

Implementation:
- Single `useEffect` that attaches one `keydown` listener on `document` using `AbortController` for cleanup (same pattern as existing WorkshopPanel.jsx lines 7-18)
- Input guard at the top of the handler: `if (tag === 'INPUT' || tag === 'TEXTAREA' || tag === 'SELECT') return;` and also `if (e.target.isContentEditable) return;`
- Switch on `e.key`:
  - `ArrowLeft`: call `e.preventDefault()` then `onPrevWeek()`
  - `ArrowRight`: call `e.preventDefault()` then `onNextWeek()`
  - `t` or `T` (use `e.key.toLowerCase() === 't'`): call `onToday()`
  - `Escape`: only if `isPanelOpen` is true, call `onClosePanel()`
  - `n` or `N` (use `e.key.toLowerCase() === 'n'`): call `onNewWorkshop()`
- `e.preventDefault()` ONLY on ArrowLeft and ArrowRight (stops page scroll). Do NOT preventDefault on T, N, or Escape.
- Dependency array: `[onPrevWeek, onNextWeek, onToday, onClosePanel, onNewWorkshop, isPanelOpen]`
- Return cleanup: `() => controller.abort()`

**2. `src/utils/slotFinder.js`**

Export a named function `findNextAvailableSlot(workshops)`.

Implementation:
- Import `startOfWeek`, `addDays`, `isSameDay`, `parseISO` from `date-fns`
- Constants: `GRID_START_HOUR = 6`, `GRID_END_HOUR = 22`
- Build a Set of occupied slot keys from non-Cancelled workshops. Key format: `${d.getFullYear()}-${d.getMonth()}-${d.getDate()}:${d.getHours()}:${d.getMinutes()}`
- Scan 7 days starting from Monday of the current week (`startOfWeek(now, { weekStartsOn: 1 })`)
- For each day, determine start scan position: if today, skip past the current time (round up to next 30-min boundary); if not today, start from GRID_START_HOUR:00
- For each 30-min slot (0 and 30 minutes per hour), check if the key is NOT in the occupied set — if so, return `{ date, hour, minute }`
- Fallback: if no slot found in 7 days, return `{ date: addDays(monday, 7), hour: 9, minute: 0 }`
- The returned `{ date, hour, minute }` matches `openCreate(date, hour, minute)` signature exactly

Follow the research code examples closely but ensure the time rounding logic for "today" is correct: if current time is e.g. 10:15, the first scanned slot should be 10:30; if 10:45, the first scanned slot should be 11:00.

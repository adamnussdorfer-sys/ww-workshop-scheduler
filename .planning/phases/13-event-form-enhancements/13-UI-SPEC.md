## Component Inventory

These are the three new UI elements added in Phase 13. All others are unchanged.

### 1. Plan Type Selector (EVNT-01)

**Pattern:** Segmented button group (two-option, single-select).

Render as two side-by-side pill buttons with a shared container row. Do NOT use radio inputs (visual inconsistency with brand). Do NOT use the `Select` dropdown — two static options warrant inline buttons.

```
[ Core ]  [ Core Plus ]
```

- Container: `flex gap-2` row, no outer border, no 59px height (compact ~40px)
- Unselected button: `px-4 py-2 text-sm font-semibold rounded-full border border-[#84ABFF] text-[#031373] bg-white hover:border-[#031373] cursor-pointer transition-colors`
- Selected button: `px-4 py-2 text-sm font-semibold rounded-full bg-ww-blue text-white border border-ww-blue cursor-pointer`
- Default value on create: `'Core'`
- Position in form: directly below the zoomType radio row, above the DateTimeRow
- Field label above the buttons: `<span className="text-[12px] font-normal text-[#031373]">Plan Type</span>`

### 2. Buffer Override Selector (EVNT-03)

**Pattern:** Segmented button group (six options, single-select).

Options: `0`, `5`, `10`, `15`, `20`, `30` (minutes). Render as a tight row of compact square/pill buttons. Default: `15` (matches `BUFFER_MINUTES` constant in conflictEngine.js).

```
[ 0 ]  [ 5 ]  [ 10 ]  [ 15 ]  [ 20 ]  [ 30 ]
```

- Container: `flex flex-wrap gap-1.5` row, no outer border
- Each option label: `{value} min` (e.g., "15 min") — omit "min" for 0: show "Off"
  - Option 0 displays as "Off" (buffer disabled — skip buffer check entirely)
  - Options 5–30 display as "5 min", "10 min", etc.
- Unselected: `px-3 py-1.5 text-[13px] font-semibold rounded-full border border-[#84ABFF] text-[#031373] bg-white hover:border-[#031373] cursor-pointer transition-colors`
- Selected: `px-3 py-1.5 text-[13px] font-semibold rounded-full bg-ww-blue text-white border border-ww-blue cursor-pointer`
- Field label above: `<span className="text-[12px] font-normal text-[#031373]">Buffer Override</span>`
- Sub-label below when value != 15: `<span className="text-[11px] text-[#031373]/60 mt-1">Default is 15 min</span>` — only shown when non-default selected
- Position in form: after the description textarea, before the action buttons

### 3. Draft Validation Relaxation (EVNT-02)

**No new field.** This is a behavioral change only: remove the hard validation gate for required fields (coach, markets, description) when `draft.status === 'Draft'` or when the user clicks "Save Draft".

Visual changes:
- When `draft.status === 'Draft'` and the form has fields left empty (coach, markets, description), no red error borders appear on those fields — they remain in the normal "empty" state (`border-[#84ABFF]`).
- Validation red border (`border-red-400`) is only applied when the user clicks "Publish" and a required-for-publish field is blank.
- Required for any save: `title` (non-empty) + `startTime` (set). All others optional for drafts.

If title is blank when "Save Draft" is clicked, show an inline error below the title `Input`:
```
<p className="text-[12px] text-red-500 mt-1 px-1">Title is required</p>
```

If startTime is not set when "Save Draft" is clicked, show an inline error below the DateTimeRow:
```
<p className="text-[12px] text-red-500 mt-1 px-1">Date and time are required</p>
```

### bufferOverride

| State | Visual |
|-------|--------|
| 15 selected (default) | 15 min pill: filled blue. Others: outline. |
| Non-default selected | Selected pill: filled blue. Sub-label "Default is 15 min" appears below row. |
| "Off" (0) selected | 0 pill: filled blue. Sub-label shows: "Buffer check disabled for this workshop" |
| Hover unselected | Border transitions from `#84ABFF` to `#031373` |

### Conflict engine integration

When `bufferOverride` is set on a workshop, `buildConflictMap` must use `workshop.bufferOverride ?? BUFFER_MINUTES` as the effective buffer threshold for that workshop's pair comparisons. This is a data contract, not a visual contract, but the UI must pass `bufferOverride` through the data model so the engine can read it.

1. `initDraft()` default shape in `WorkshopForm.jsx`

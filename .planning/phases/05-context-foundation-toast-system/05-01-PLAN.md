<task type="auto">
  <name>Task 1: Install sonner and extend App.jsx with Toaster, filters state, and memoized context</name>
  <files>package.json, src/App.jsx</files>
  <action>
1. Install sonner: `npm install sonner`

2. Edit `src/App.jsx`:
   - Add imports: `import { useMemo } from 'react'` (add to existing useState import), `import { Toaster, toast } from 'sonner'`
   - Add filters state after workshops state:
     ```js
     const [filters, setFilters] = useState({
       coaches: [],
       types: [],
       statuses: [],
       markets: [],
     })
     ```
   - Replace the inline `value={{ coaches, setCoaches, workshops, setWorkshops }}` with a `useMemo`-wrapped context value:
     ```js
     const contextValue = useMemo(
       () => ({ coaches, setCoaches, workshops, setWorkshops, filters, setFilters, toast }),
       [coaches, workshops, filters]
     )
     ```
     Note: `setCoaches`, `setWorkshops`, `setFilters` are stable React setState refs — omit from deps array. `toast` is a module-level singleton — also stable.
   - Change Provider to `<AppContext.Provider value={contextValue}>`
   - Add `<Toaster>` as first child inside `<AppContext.Provider>`, before `<AppShell>`:
     ```jsx
     <Toaster
       position="bottom-right"
       duration={3500}
       visibleToasts={3}
       expand={false}
       toastOptions={{
         style: {
           fontFamily: 'Inter, system-ui, sans-serif',
           fontSize: '14px',
         },
       }}
     />
     ```
     CRITICAL: `<Toaster>` must be outside `<Routes>` and outside `<AppShell>` to survive page navigation. Place it as a sibling before `<AppShell>`.

3. Do NOT add richColors — WW brand colors (blue success, coral error) don't match sonner's semantic green/red. Default unstyled is correct.

4. Do NOT create a toasts useState array — sonner manages its own state via observer pattern. The `toast` on context is just a re-export for convenience.
  </action>

**src/components/calendar/WorkshopCard.jsx** — Two changes:

1. **INTR-06 — Hover lift**: Replace the outer div's className. Remove `hover:brightness-95` and `transition-all`. Add `transition-[transform,box-shadow] duration-150 ease-out hover:-translate-y-0.5 hover:shadow-md`. Add `motion-reduce:transition-none motion-reduce:hover:transform-none` for accessibility. The result:
```jsx
className={`h-full overflow-hidden rounded text-xs px-1.5 py-1 cursor-pointer relative
  transition-[transform,box-shadow] duration-150 ease-out
  hover:-translate-y-0.5 hover:shadow-md
  motion-reduce:transition-none motion-reduce:hover:transform-none
  ${cardStyle} ${ringClass}${isFiltered ? ' opacity-25 pointer-events-none' : ''}`}
```

Do NOT use `translate-y` via margin/top — only CSS `transform: translateY()` (which is what Tailwind's `-translate-y-0.5` compiles to). This is GPU-composited and does not trigger layout reflow. The click target is unchanged since transform does not affect layout box.

2. **INTR-08 — Conflict pulse**: Add `animate-conflict-pulse` to the AlertTriangle icon wrapper div (NOT the status dot — the status dot indicates Published/Draft status, while AlertTriangle is the conflict indicator). Gate on `hasConflicts` which is already computed. Add `motion-reduce:animate-none`.

Change the AlertTriangle wrapper from:
```jsx

```
to:

Note: `hasConflicts` is always true when this div renders (it's inside an `{hasConflicts && ...}` conditional), so the ternary is technically always true here. But keep the explicit gate for clarity and to not break if the conditional structure ever changes.

```jsx
const weekWorkshopsCount = useMemo(() => {
  return weekDays.reduce((count, day) => {
    return count + workshops.filter(

}, [workshops, weekDays]);
```

**Create `src/utils/availabilityBands.js`:**

Create a pure utility module that computes availability band positions for a given day. This file also owns the grid constants (GRID_START_HOUR, GRID_END_HOUR, PX_PER_HOUR, PX_PER_MIN, GRID_HEIGHT) which CalendarGrid will later import from here to avoid duplication.

Export `getAvailabilityBands(coaches, day)`:
- Takes the coaches array and a Date object for a single day
- Converts the day to lowercase day name using `format(day, 'EEEE').toLowerCase()` from date-fns (same pattern as `coachAvailability.js`)
- Iterates each coach:
  - Skip if `coach.status === 'inactive'`
  - Find all availability entries where `a.day === dayName`
  - For each matching slot, parse `start`/`end` HH:MM strings to minutes
  - Clamp to grid bounds: `clampedStart = Math.max(startMins, GRID_START_HOUR * 60)`, `clampedEnd = Math.min(endMins, GRID_END_HOUR * 60)`
  - Skip if `clampedEnd <= clampedStart`
  - Compute `top = (clampedStart - GRID_START_HOUR * 60) * PX_PER_MIN`
  - Compute `height = (clampedEnd - clampedStart) * PX_PER_MIN`
  - Push `{ coachId: coach.id, coachIndex: idx, top, height }` to result array
- Return the array of bands

3. **Import `useApp`** from `../../context/AppContext` to access `filters` for filter-scoped overlay (per research recommendation). When a coach filter is active (`filters.coaches.length > 0`), only show bands for those coaches.

4. **Compute availability bands with useMemo:**
   ```js
   const { filters } = useApp();

   const availabilityBands = useMemo(() => {
     if (!showOverlay) return new Map();
     const result = new Map();
     // When coach filter active, only show filtered coaches' bands
     const overlayCoaches = filters.coaches.length > 0
       ? coaches.filter(c => filters.coaches.includes(c.id))
       : coaches;
     weekDays.forEach(day => {
       result.set(day.toISOString(), getAvailabilityBands(overlayCoaches, day));
     });
     return result;
   }, [showOverlay, coaches, weekDays, filters.coaches]);
   ```
   Short-circuits when overlay is off (returns empty Map). This prevents the 126-band computation on every render.

   IMPORTANT: When passing a filtered subset of coaches to `getAvailabilityBands`, the `coachIndex` inside the utility will be relative to the filtered array. This is fine — it only controls which color is picked from COACH_OVERLAY_COLORS. For consistent coach-to-color mapping regardless of filter state, use a stable index from the original coaches array. To achieve this, modify the useMemo to pass the full coaches array but with a filter set:

   ```js
   const availabilityBands = useMemo(() => {
     if (!showOverlay) return new Map();
     const result = new Map();
     const coachFilterSet = filters.coaches.length > 0
       ? new Set(filters.coaches)
       : null;
     weekDays.forEach(day => {
       const bands = getAvailabilityBands(coaches, day)
         .filter(band => !coachFilterSet || coachFilterSet.has(band.coachId));
       result.set(day.toISOString(), bands);
     });
     return result;
   }, [showOverlay, coaches, weekDays, filters.coaches]);
   ```
   This approach computes all bands (preserving stable coachIndex for color) then filters to active coaches.

5. **Render bands inside each day column**, placed AFTER slot lines and BEFORE saturation bars and workshop cards. This puts them at the bottom of the z-order via DOM order:

   ```jsx
   {/* Availability overlay bands — rendered before workshop cards, pointer-events-none */}
   {showOverlay && availabilityBands.get(day.toISOString())?.map((band) => (
     <div
       key={`avail-${band.coachId}`}
       className={`absolute left-0 right-0 pointer-events-none ${COACH_OVERLAY_COLORS[band.coachIndex % COACH_OVERLAY_COLORS.length]}`}
       style={{ top: band.top, height: band.height, zIndex: 0 }}
       aria-hidden="true"
     />
   ))}
   ```

   Place this JSX block immediately after the SLOT_LINES `.map()` block (after the closing `})}` of slot lines) and before the saturation bars block. The `zIndex: 0` ensures bands sit below workshop cards (which start at `zIndex: 1`). `pointer-events-none` ensures clicks pass through to workshop cards and day column slot handler.

6. **Verify no interference with existing functionality:**
   - Workshop card clicks must still work (pointer-events-none on bands)
   - Day column click-to-create must still work (bands don't intercept)
   - Filter dim effect still applies to workshop cards
   - Saturation bars still visible above bands

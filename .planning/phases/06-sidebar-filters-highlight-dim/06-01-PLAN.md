  <name>Task 1: Create filterEngine.js pure utility</name>
  <files>src/utils/filterEngine.js</files>
  <action>
Create `src/utils/filterEngine.js` following the same module pattern as `conflictEngine.js` (pure functions, no React imports, named exports).

**Export 1: `getMatchedWorkshopIds(workshops, filters)`**
- Takes full workshops array and filters object `{ coaches: string[], types: string[], statuses: string[], markets: string[] }`
- Returns a `Set<string>` of workshop IDs that match ALL active filter dimensions
- Logic: OR within a dimension, AND across dimensions
  - If a dimension array is empty, all workshops pass that dimension (no filter applied)
  - Coach dimension: workshop passes if `ws.coachId` is in `filters.coaches` OR `ws.coCoachId` is in `filters.coaches` (handle null coCoachId)
  - Type dimension: `filters.types.includes(ws.type)`
  - Status dimension: `filters.statuses.includes(ws.status)`
  - Market dimension: `ws.markets.some(m => filters.markets.includes(m))` (markets is always an array on workshops)
- When ALL dimensions are empty (no filters active), return a Set of ALL workshop IDs (all match)

**Export 2: `hasActiveFilters(filters)`**
- Returns boolean: true if any dimension array has length > 0
- Used by consumers to know whether to apply dim effect and show filter pills

Use JSDoc comments matching the conflictEngine.js style. Do not import React or any external libraries — this is a pure JS utility.

**toggleFilter helper (inside component body):**
```js
function toggleFilter(dimension, value) {
  setFilters(prev => {
    const current = prev[dimension];
    const next = current.includes(value)
      ? current.filter(v => v !== value)
      : [...current, value];
    return { ...prev, [dimension]: next };
  });
}
```

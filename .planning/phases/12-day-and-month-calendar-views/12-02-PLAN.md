  <action>
Create `src/components/calendar/MonthView.jsx` — a month grid component.

Props: `{ currentDate, workshops, coaches, conflictMap, onWorkshopClick, onDayClick, filteredIds, anyFilterActive }`

**Date math:**
- Import from date-fns: `startOfMonth, endOfMonth, startOfWeek, endOfWeek, addDays, isSameMonth, isSameDay, isToday, format, parseISO, getDay`
- Compute `monthStart = startOfMonth(currentDate)`, `monthEnd = endOfMonth(currentDate)`
- Compute `calendarStart = startOfWeek(monthStart, { weekStartsOn: 1 })` — this is the Monday of the first visible week
- Compute `calendarEnd = endOfWeek(monthEnd, { weekStartsOn: 1 })` — this is the Sunday of the last visible week
- Generate array of weeks: iterate from calendarStart by adding 7 days until past calendarEnd. Each week is an array of 7 Date objects (Mon-Sun).
- This produces a 5 or 6 row grid depending on the month.

**Layout:**
- Column headers: Mon, Tue, Wed, Thu, Fri, Sat, Sun — styled same as CalendarGrid (uppercase, 10px, text-slate-600)
- Grid: `grid grid-cols-7` with `border border-border rounded-lg overflow-hidden bg-white`
- Each day cell: fixed min-height (e.g., `min-h-[120px]`), border-t + border-l, padding `p-1.5`

**Day cell content:**
1. **Date number:** top-right or top-left of cell. If `isToday(day)`: render in a small circle with `bg-ww-blue text-white rounded-full w-6 h-6 flex items-center justify-center text-xs font-bold`. If outside current month (`!isSameMonth(day, currentDate)`): `text-slate-300`. Otherwise: `text-ww-navy text-xs font-semibold`.

2. **Workshop pills:** Filter `workshops` to those matching this day (non-Cancelled, `isSameDay(parseISO(ws.startTime), day)`). Show up to 3 pills. Each pill is a small horizontal bar:
   - Height: `h-5` with `text-[10px] leading-tight px-1 rounded truncate cursor-pointer`
   - Type-colored left border (4px): use same color mapping as WorkshopCard TYPE_CARD_STYLES but adapted for pills. Define a `TYPE_PILL_STYLES` lookup:
     - 'Weekly Connection': `'bg-blue-50 border-l-2 border-ww-blue'`
     - 'All In': `'bg-purple-50 border-l-2 border-purple-600'`
     - 'Special Event': `'bg-rose-50 border-l-2 border-ww-coral'`
     - 'Coaching Corner': `'bg-teal-50 border-l-2 border-teal-500'`
     - 'Movement/Fitness': `'bg-green-50 border-l-2 border-green-500'`
   - Title text truncated: `workshop.title`
   - `onClick` on pill: call `onWorkshopClick(ws.id)` with `e.stopPropagation()` to prevent day cell click
   - If `anyFilterActive && !filteredIds.has(ws.id)`: add `opacity-25 pointer-events-none` (same dim pattern as week view)

3. **Overflow link:** If day has more than 3 workshops, show `"+N more"` where N is total - 3. Style: `text-[10px] text-ww-blue hover:underline cursor-pointer mt-0.5`. `onClick`: call `onDayClick(day)` to drill into day view.

4. **Conflict dot:** If any workshop on this day has conflicts (check via `conflictMap`), render a small red dot in the top-right corner of the cell: `<span className="absolute top-1 right-1 w-2 h-2 rounded-full bg-red-500" />`. Cell needs `relative` positioning.

**Cell click:** The entire day cell is clickable — `onClick={() => onDayClick(day)}`. Workshop pill clicks use `e.stopPropagation()` to avoid triggering day click.

**Cell hover:** Add subtle hover effect per Claude's discretion: `hover:bg-surface-2/50 transition-colors`.

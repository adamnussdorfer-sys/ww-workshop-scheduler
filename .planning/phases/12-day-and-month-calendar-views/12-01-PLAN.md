**1. Create `src/components/calendar/DayView.jsx`:**

A single-day hour grid component following the same visual structure as CalendarGrid but with one column filling the full width.

Props: `{ date, workshops, coaches, conflictMap, onWorkshopClick, onSlotClick, filteredIds, anyFilterActive, showOverlay }`

Implementation:
- Import and reuse the shared grid constants from `availabilityBands.js`: `GRID_START_HOUR`, `GRID_END_HOUR`, `PX_PER_HOUR`, `PX_PER_MIN`, `GRID_HEIGHT`, `getAvailabilityBands`, `COACH_OVERLAY_COLORS`
- Import `WorkshopCard` from `./WorkshopCard`
- Import `getSaturatedSlots` from `../../utils/conflictEngine`
- Import `useApp` from `../../context/AppContext` (for `filters.coaches` in overlay)
- Import date-fns: `parseISO, getHours, getMinutes, differenceInMinutes, isSameDay, format, isToday`

Header: Full day name + date (e.g., "Monday, March 9, 2026") — use `format(date, 'EEEE, MMMM d, yyyy')`. If today, add a subtle accent indicator (e.g., `text-ww-blue font-bold`).

Body layout: Same time gutter (w-16) on the left as CalendarGrid. Single day column fills all remaining width (no 7-col grid — just one `flex-1` column).

Reuse `getEventPosition` helper (copy from CalendarGrid or extract — simplest is to define inline since it's 6 lines). Same slot row lines pattern (32 lines at 30-min intervals). Same click-to-create slot handler with scroll offset logic from CalendarGrid.

Workshop cards: Since only one column, cards can be wider. Use `left: 2, right: 2` positioning (no `idx * 4` stagger unless overlapping — use same idx stagger for overlap cases). Cards are rendered using `WorkshopCard` with the same props pattern as CalendarGrid. Because cards are wider, WorkshopCard already shows title/coach/time — the wider space naturally shows more text since `truncate` won't clip as early.

Availability overlay: Same pattern as CalendarGrid — if `showOverlay`, compute bands for this single day using `getAvailabilityBands(coaches, date)`, filter by `filters.coaches` if active. Render as absolute-positioned `pointer-events-none` divs with `COACH_OVERLAY_COLORS`.

Saturation bars: Same pattern as CalendarGrid — compute `getSaturatedSlots` for this day's workshops.

Filter dim: Same pattern — if `anyFilterActive && !filteredIds.has(ws.id)`, apply `opacity-25 pointer-events-none` on the card wrapper.

**2. Refactor `src/pages/ScheduleCalendar.jsx`:**

Replace `currentWeekStart` with a more general `currentDate` state initialized to `new Date()`. Derive `currentWeekStart` from `currentDate` using `startOfWeek(currentDate, { weekStartsOn: 1 })` so the week view still works identically.

Add date-fns imports: `addDays, subDays, startOfMonth, endOfMonth, addMonths, subMonths` (some may already be imported).

Navigation callbacks — make view-mode-aware:
- `prevPeriod`: week mode = `subWeeks(d, 1)`, day mode = `subDays(d, 1)`, month mode = `subMonths(d, 1)`
- `nextPeriod`: week mode = `addWeeks(d, 1)`, day mode = `addDays(d, 1)`, month mode = `addMonths(d, 1)`
- `goToToday`: always `new Date()` regardless of view mode
- `isCurrentPeriod`: week = `isSameWeek`, day = `isSameDay(currentDate, new Date())`, month = use `isSameMonth` from date-fns

Header text — adapt per view mode:
- Week: existing `"Week of MMM d - MMM d, yyyy"` format
- Day: `format(currentDate, 'EEEE, MMMM d, yyyy')` e.g., "Monday, March 9, 2026"
- Month: `format(currentDate, 'MMMM yyyy')` e.g., "March 2026"

Nav bar arrow aria-labels: adapt per view mode ("Previous day"/"Next day", "Previous week"/"Next week", "Previous month"/"Next month").

Availability overlay button: show in week and day views, hide in month view (per CONTEXT.md: "No availability overlay in month view"). Conditionally render the Eye/EyeOff button: `{viewMode !== 'month' && (...)}`.

Day label drill-down in week view header: This is inside CalendarGrid. Pass a new prop `onDayClick` to CalendarGrid. In CalendarGrid's day header, make the day number a clickable element. When clicked, call `onDayClick(day)`. In ScheduleCalendar, the `onDayClick` handler sets `viewMode` to `'day'` and `currentDate` to the clicked day.

Create Workshop button behavior: In day view, instead of `openNewWithNextSlot`, pre-fill from the current day — use `openCreate(currentDate, nextHour, 0)` or still use `findNextAvailableSlot` (keeping existing behavior is fine since the slot finder already picks the next available slot).

Filter empty states for day view:
- `anyFilterActive && dayMatchCount === 0`: "No matching workshops on [Day, Date]" + Clear filters CTA
- `!anyFilterActive && dayWorkshopsCount === 0`: "No workshops on [Day, Date]" + Create Workshop CTA

Compute `dayMatchCount` and `dayWorkshopsCount` (similar to existing weekMatchCount/weekWorkshopsCount) when `viewMode === 'day'`.

Render the DayView component when `viewMode === 'day'`, passing: `date={currentDate}`, `workshops`, `coaches`, `conflictMap`, `onWorkshopClick={openWorkshop}`, `onSlotClick={openCreate}`, `filteredIds`, `anyFilterActive`, `showOverlay`.

Keep the existing "Month view coming soon" placeholder — Plan 02 will replace it.

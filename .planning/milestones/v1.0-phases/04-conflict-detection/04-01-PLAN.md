Create `src/utils/conflictEngine.js` with these exports:

**`buildConflictMap(workshops, coaches)`** — Returns a `Map<workshopId, ConflictResult>` where:
```
ConflictResult = {
  conflicts: Conflict[],   // array of conflict objects
  ringColor: 'red' | 'orange' | null,
  hasConflicts: boolean
}

Conflict = {
  type: 'double-booking' | 'buffer' | 'availability',
  severity: 'red' | 'orange',
  message: string    // human-readable one-liner for panel display
}
```

Implementation steps:
1. Filter to `active = workshops.filter(w => w.status !== 'Cancelled')` — per CONFLICT-05, cancelled workshops excluded
2. Build `coachMap = new Map(coaches.map(c => [c.id, c]))` for O(1) name lookups
3. Initialize result Map with empty entries for all active workshops
4. **Double-booking detection (CONFLICT-01):** O(n^2) pairwise loop over active workshops. For each pair, compute shared coaches using `getSharedCoaches(a, b)` which checks all combinations of coachId/coCoachId. If shared coaches exist AND `timesOverlap(a.startTime, a.endTime, b.startTime, b.endTime)` (using `new Date(start) < new Date(otherEnd) && new Date(end) > new Date(otherStart)`), add a conflict to BOTH workshops with severity 'red' and message: `Coach ${coachName} also assigned to "${otherTitle}" at ${formatTime(otherStart)}`
5. **Buffer violation detection (CONFLICT-03):** Group workshops by coach (primary AND co-coach) using `groupByCoach(active)`. For each coach's workshops, sort by startTime. Check adjacent pairs: if NOT overlapping (skip overlaps — handled by double-booking) AND `differenceInMinutes(b.startTime, a.endTime) < 15`, add buffer conflict to both with severity 'orange' and message: `Only ${gap} min gap before/after "${otherTitle}" — recommend 15+ min buffer`
6. **Availability conflict detection (CONFLICT-04):** For each active workshop, check both coachId and coCoachId. Import and call `getCoachAvailability(coach, date, hour, minute)` from `./coachAvailability`. If not available, add conflict with severity 'orange' and message: `Coach ${name} not available: ${reason}`
7. After all conflicts computed, derive ringColor for each entry: 'red' if any red conflicts, 'orange' if any orange, null if none. Set hasConflicts flag.

**`getSaturatedSlots(dayWorkshops)`** — For CONFLICT-02:
- Takes an array of workshops for a single day (already filtered to non-cancelled)
- Iterates 32 slots (0-31) where each slot covers 30 min starting at `GRID_START_HOUR * 60 + i * 30`
- Counts workshops whose time range overlaps each slot
- Returns `Array<{slotIndex: number, count: number}>` for slots with count >= 4 (SATURATION_THRESHOLD = 4)
- Convert workshop times to minutes-from-midnight using `new Date(isoString).getHours() * 60 + new Date(isoString).getMinutes()`

Helper functions (internal, not exported):
- `timesOverlap(startA, endA, startB, endB)` — `new Date(startA) < new Date(endB) && new Date(endA) > new Date(startB)`
- `getSharedCoaches(a, b)` — returns array of coach IDs appearing in both workshops' [coachId, coCoachId]
- `groupByCoach(workshops)` — returns Map<coachId, workshop[]> grouping by both primary and co-coach
- `timeToMinutes(isoString)` — hours*60 + minutes from ISO string
- `formatTime(isoString)` — `format(parseISO(isoString), 'h:mm a')` using date-fns
- `addConflict(resultMap, workshopId, conflict)` — pushes conflict, updates ringColor immediately

Use `import { parseISO, differenceInMinutes, format, getHours, getMinutes } from 'date-fns'` — all already in the project.
Use `import { getCoachAvailability } from './coachAvailability'` — reuse Phase 3 utility exactly as built.

CONFLICT-05: No blocking logic. Functions return data only, never prevent save/publish. This is intrinsically satisfied by the pure-function design — the engine has no side effects.

Run tests until all pass. If any test needs minor adjustment for message format, adjust the test to match the implementation's actual format (the message content matters less than the structure).
